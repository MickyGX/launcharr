# Changelog

## [v0.2.54] - 2026-03-30

### Added

- Support for hosting Launcharr under a base path (for example `/launcharr`), including route mounting, redirects, and public URL generation.

### Fixed

- Local and Plex login flows now work when Launcharr is served under a base path.
- Service worker and manifest assets now resolve correctly under a base path.
- Icon assets referenced by overview styles now load correctly under a base path.

### Changed

- Base-path handling now rewrites root-absolute asset, link, and API URLs at runtime to match the configured subpath.

### Migration Notes

- If you host Launcharr at a subpath, set **Settings → General → Base URL Path** to that path (for example `/launcharr`) and access the app via that URL.
- Reverse proxies should forward the subpath without stripping it.

### Verification Checklist

- [ ] Access the app at the configured base path (for example `/launcharr`) and confirm login works.
- [ ] Plex SSO completes successfully under the base path.
- [ ] Static assets (icons, CSS, JS, manifest, service worker) load without 404s.
- [ ] Dashboard and app overview pages render correctly under the base path.
