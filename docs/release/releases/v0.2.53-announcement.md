# Launcharr Release Announcement

## Headline

Launcharr `v0.2.53` is now available

## Short Summary

Launcharr `v0.2.53` is a bug-fix release resolving two regressions introduced in v0.2.52: local user management and custom HTTP header persistence are both fully restored.

## What's New

- **Local user creation and password changes fixed** — adding a local user or changing a local user's password no longer throws a server error. The `hashPassword` function was missing from the settings route context, causing a `ReferenceError` on any user write operation.
- **Custom HTTP headers now persist** — headers entered in Settings → Apps → Connection are saved correctly. A mismatch between form field names (`customHeaderName[]`) and the server's expected key (`customHeaderName`) meant all headers were silently discarded on save.

## Why It Matters

- Installations relying on local (non-Plex) user accounts were blocked from adding new users or updating passwords since v0.2.52.
- Anyone configuring custom HTTP headers (e.g. for Cloudflare Access or reverse-proxy auth tokens) would find them missing after every save, requiring manual edits to `config.json` as a workaround.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.53`
- Supplementary tags: `mickygx/launcharr:development`, `mickygx/launcharr:latest`
- No migration steps required.
- If you previously entered custom HTTP headers that failed to save, re-enter them after upgrading — they will persist correctly from v0.2.53 onwards.
