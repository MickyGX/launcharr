# Changelog

## [v0.2.51] - 2026-03-22

### Changed

- Reduced background widget load for Arr apps by switching Launcharr's internal widget monitor to lightweight system-status checks for Radarr, Sonarr, Lidarr, and Readarr.
- Added cached Arr summary metrics so Launcharr can keep useful widget counts available without repeatedly pulling full library datasets during routine monitor ticks.
- Tightened local packaging hygiene so local-only development modules, temporary compose overrides, and dev notes stay out of git history and Docker build context.

### Fixed

- Fixed public image packaging drift so local-only development content is no longer shipped accidentally in release builds.
- Improved widget responsiveness for Arr-backed services with larger libraries by avoiding unnecessary full-library fetches in the internal monitor path.

### Migration Notes

- No schema migration required.
- Existing installs should hard refresh browser assets after upgrade so the updated widget runtime is loaded.
- Operators using Launcharr with large Arr libraries should see lower background widget polling impact after upgrade.

### Verification Checklist

- [ ] Internal widget monitoring for Radarr, Sonarr, Lidarr, and Readarr succeeds when each app is reachable.
- [ ] Arr widgets still show summary metrics after normal dashboard use populates the cache.
- [ ] Release builds do not include local-only development module content.
- [ ] Browser assets hard refresh cleanly after upgrade.
