# Launcharr Release Announcement

## Headline

Launcharr `v0.2.40` is now available

## Short Summary

Launcharr `v0.2.40` adds a guided admin `Quick Start Guide`, introduces a release `Welcome` popup powered by changelog highlights, and hardens frontend cache-busting to reduce stale assets after upgrades.

## What's New

- Added an admin-only `Quick Start Guide` modal with setup steps and a shortcut to `Settings -> Custom -> Sidebar`.
- Added a release `Welcome` modal that surfaces highlights from `Added` / `Changed` / `Fixed` sections for the installed version.
- Added backend version API support for version-specific release links and changelog highlights.
- Updated frontend asset URLs to include runtime `assetVersion` cache-busting query parameters across key views.
- Automatically clears pending onboarding prompts when active apps already exist.

## Why It Matters

- New installs get a clearer path to restoring defaults, configuring apps, and building dashboard cards.
- Release highlights are now surfaced in-app, making new-version changes easier to discover.
- Upgrades are more reliable because browsers are less likely to serve stale scripts/styles after deploys.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.40`
- Supplementary tag: `mickygx/launcharr:development` (if selected during release flow).
- No migration steps required.
- A hard refresh is usually no longer needed, but may help once if a previous service worker cached older assets.

## Documentation

- Changelog: `docs/release/releases/v0.2.40.md`

## Known Issues

- The `Quick Start Guide` is admin-only and depends on onboarding state; if you are testing on an existing populated install, it may auto-dismiss by design once active apps are detected.
