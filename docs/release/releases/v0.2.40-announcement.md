# Launcharr Release Announcement

## Headline

Launcharr `v0.2.40` is now available

## Short Summary

Launcharr `v0.2.40` adds multi-dashboard support with per-dashboard visibility and saved layouts, plus admin role-view switching so you can preview guest/user/co-admin experiences without leaving your account.

## What's New

- Added multiple dashboards (up to `10`) with dashboard tabs in `Settings -> Custom -> Dashboard`.
- Added dashboard create/duplicate/delete controls, including duplicate-from-existing and blank dashboard creation.
- Added per-dashboard metadata and access controls:
  - Dashboard name
  - Dashboard icon
  - Dashboard role visibility (multi-role selection)
- Added per-dashboard saved dashboard state so card layout/configuration can differ between dashboards.
- Added dashboard-aware sidebar navigation and `?dashboard=` routing across dashboard/app/settings pages.
- Added admin role-view picker (`guest`, `user`, `co-admin`, `admin`) with dashboard-availability indicators and smarter redirect fallback.
- Updated asset URLs on affected pages with runtime cache-busting query params to reduce stale CSS/JS after upgrades.

## Why It Matters

- You can maintain separate dashboards for different use cases (for example, family, admin tools, or focused media views) without reconfiguring one shared layout.
- Role-based dashboard visibility pairs with admin role-view switching, making it much easier to validate what each user role can actually see.
- Dashboard navigation is more predictable because Launcharr preserves the selected dashboard across pages.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.40`
- Supplementary tag: `mickygx/launcharr:development` (if selected during release flow).
- No migration steps required.
- Existing installs continue using the default `main` dashboard until you add more.
- A hard refresh may still help once if your browser/service worker has older cached assets.

## Known Issues

- Dashboard role visibility can intentionally leave a role with no visible dashboard; in that case Launcharr redirects to the next accessible page for that role.
