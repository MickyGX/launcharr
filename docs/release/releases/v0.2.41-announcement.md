# Launcharr Release Announcement

## Headline

Launcharr `v0.2.41` is now available

## Short Summary

Launcharr `v0.2.41` is a maintenance and hardening release with a Romm iframe launch fix, local login rate limiting, better session-secret defaults, and major internal route/view/CSS refactors backed by new auth tests.

## What's New

- Fixed Romm iframe viewer launch cookie priming for more reliable embedded Romm launches.
- Added local login rate limiting (`POST /login`) with temporary lockout and clearer error messaging after repeated failures.
- Replaced the insecure default session-secret fallback with runtime-generated secrets plus an operator warning when `SESSION_SECRET` is unset.
- Refactored route registration into `src/routes/*` modules and extracted shared EJS partials to reduce duplication and improve maintainability.
- Split frontend CSS into page-focused bundles (`styles-auth`, `styles-home`, `styles-layout`, `styles-overview`, `styles-settings`) while keeping shared tokens in `styles.css`.
- Added automated auth and route-guard tests (`npm test`) using `supertest`.

## Why It Matters

- Admins get safer default behavior for sessions and basic protection against repeated local-login brute-force attempts.
- Romm iframe launches are more reliable in environments that require cookie priming.
- The internal refactors reduce duplication and make future feature work/bug fixes easier to ship safely.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.41`
- Supplementary tag: `mickygx/launcharr:development` (if selected during release flow).
- No migration steps required.
- Set `SESSION_SECRET` explicitly if you want sessions to survive restarts.
- A hard refresh may help once after upgrade due to the CSS asset split.
