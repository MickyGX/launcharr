# Launcharr Release Announcement

## Headline

Launcharr `v0.2.42` is now available

## Short Summary

Launcharr `v0.2.42` delivers a major widget release: expanded System Info widgets, improved widget editor UX/layout behavior, and new delayed widget status notifications via Apprise.

## What's New

- Expanded System Info widgets with configurable CPU, RAM, multi-disk, search, and weather modules.
- Added `Show Total Space (RAM + Disks)` support for System Info widget displays.
- Added widget status notifications with explicit settings toggles plus delay threshold controls.
- Added a background widget-state monitor that detects sustained online/offline transitions and notifies through Apprise.
- Added a live widget monitor debug line in `Settings -> Log` and `GET /api/widget-status-monitor` for real-time transition visibility.
- Updated widget settings UX with compact up/down row insertion controls, improved add-menu layering, and better dashboard/settings visual parity.
- Added/updated widget-specific icon and runtime assets (`public/widgets-stat-cards.js`, weather/system icons).

## Why It Matters

- Admins can now build richer, cleaner system telemetry cards directly in Launcharr widgets without custom tooling.
- Notification noise is reduced by delay-based transition detection, while still surfacing meaningful app state changes quickly.
- Widget editing is faster and more predictable across desktop and mobile, and debug visibility makes notification behavior easier to validate.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.42`
- Supplementary tag: `mickygx/launcharr:development` (if selected during release flow).
- No migration steps required.
- Rebuild/restart Launcharr, then hard refresh once to ensure the new widget script/style assets are loaded.
- For widget status notifications, ensure both Apprise and widget-status toggles are enabled in `Settings -> Notifications`.
