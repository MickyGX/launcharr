# Launcharr Release Announcement

## Headline

Launcharr `v0.2.52` is now available

## Short Summary

Launcharr `v0.2.52` adds multi-arch Docker CI builds and per-app custom HTTP header support for reverse-proxy and auth header flexibility.

## What's New

- Multi-arch Docker images (`linux/amd64` + `linux/arm64`) now publish automatically via GitHub Actions on every release tag.
- Per-app custom HTTP headers: add any extra request headers to an app in Settings → Apps → Connection — they're merged into every outbound call Launcharr makes for that app.

## Why It Matters

- ARM64 users (Raspberry Pi, Apple Silicon NAS, Synology, QNAP ARM models) can now pull a native image instead of running an emulated `amd64` container.
- Users behind reverse proxies that require custom auth or forwarding headers no longer need a workaround — the headers travel with every request automatically.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.52`
- Supplementary tags: `mickygx/launcharr:development`, `mickygx/launcharr:latest`
- No migration steps required.
- Custom headers are opt-in and empty by default — no existing configuration is changed on upgrade.
