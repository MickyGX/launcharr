# Launcharr Release Announcement

## Headline

Launcharr `v0.2.43` is now available

## Short Summary

Launcharr `v0.2.43` focuses on app-management improvements: broader built-in integrations, multi-instance defaults, new specialty overview modules, security hardening, and fixes for custom/default app add flows.

## What's New

- Expanded built-in app catalog/category/icon coverage with updated defaults and docs.
- Added multi-instance metadata support across default apps.
- Added overview endpoints/modules for MeTube, Audiobookshelf, Tdarr, Immich, Wizarr, Uptime Kuma, Guacamole, and Traefik.
- Added app behavior controls in `Settings -> Apps -> General` and moved add-default/add-custom controls to the top of that page.
- Added security hardening: conditional trust proxy, CSRF protections, baseline headers, tightened payload limits, stronger local password policy.

## Why It Matters

- App onboarding and customization are faster and more reliable from a single always-visible app-general surface.
- Specialty app overview support is broader, reducing context switching into external dashboards.
- Security baseline is stronger out-of-the-box for reverse-proxy and local-auth deployments.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.43`
- Supplementary tag: `mickygx/launcharr:development`
- No migration steps required.
- If you run behind a reverse proxy, set:
  - `TRUST_PROXY=true`
  - `TRUST_PROXY_HOPS=1`
  - optional `COOKIE_SECURE=true` for HTTPS-only domain usage
- Rebuild/restart Launcharr, then hard refresh browser assets once.
