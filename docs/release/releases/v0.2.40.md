# Changelog

## [v0.2.40] - 2026-02-24

### Added

- Added multi-dashboard definitions (`config.dashboards`) with a default `main` dashboard plus support for additional dashboards (up to `10` total).
- Added admin dashboard management in `Settings -> Custom -> Dashboard`:
  - Dashboard tabs
  - `+ Dashboard` create/duplicate flow (blank or copy from an existing dashboard)
  - `- Dashboard` delete flow for non-main dashboards
  - Per-dashboard name and icon editing
  - Per-dashboard role visibility (multi-select)
- Added per-dashboard state snapshots so dashboard-specific card layouts/settings are stored independently (including dashboard card order/remove state, combined-card settings/order, and dashboard widget state).
- Added dashboard-aware sidebar navigation entries across dashboard/app/settings pages, including sub-links for secondary dashboards and `?dashboard=` routing.
- Added admin role-view switching for `guest`, `user`, `co-admin`, and `admin` via `/switch-view`, with user-menu role picker controls and per-role dashboard availability indicators.

### Changed

- Updated dashboard/app/settings route rendering to resolve an active dashboard per request and apply that dashboard's saved state snapshot before building dashboard content/settings lists.
- Updated dashboard save/add/remove handlers to persist changes into the selected dashboard context instead of always mutating the global main dashboard state.
- Updated role-switch redirect behavior to preserve the current page when the target role can access it, with dashboard-aware fallback to the first visible dashboard or next accessible page.
- Updated dashboard visibility handling to support explicit per-role dashboard visibility lists (while retaining legacy compatibility fields).

### Fixed

- Fixed version badge admin-view detection so version/status UI behavior remains correct when using the new role-view picker links.
- Reduced stale frontend asset issues after upgrades by adding runtime cache-busting query params (`assetVersion`) to updated CSS/JS includes across dashboard/app/settings pages.
- Improved role-view navigation fallback when a selected dashboard or page is not accessible for the emulated role.

### Migration Notes

- No schema/data migration required.
- Existing installs continue to use the default `main` dashboard automatically until additional dashboards are created.
- The default `Dashboard` (`main`) cannot be deleted. Additional dashboards can be deleted from dashboard settings.
- Review dashboard visibility roles if you plan to use admin role-view switching to validate guest/user/co-admin experiences.
