# Changelog

## [v0.2.40] - 2026-02-24

### Added

- Added admin onboarding `Quick Start Guide` API endpoints (`GET /api/onboarding/quick-start`, `POST /api/onboarding/quick-start/dismiss`) plus a guided modal with setup steps and a direct link into `Settings -> Custom -> Sidebar`.
- Added a release `Welcome` modal in the version/status area that surfaces highlights from the installed version's changelog and links to release notes / dashboard settings.
- Added changelog highlight parsing for `Added` / `Changed` / `Fixed` sections so `/api/version` can return release highlights and a version-specific release-notes URL.

### Changed

- Updated the version badge flow to show onboarding first and only show release welcome when onboarding is not currently displayed.
- Updated release welcome tracking to persist per version in browser storage (`launcharr-release-welcome-seen`) so users only see a release once.
- Updated rendered asset URLs across dashboard, overview, launch, and settings views to use a runtime `assetVersion` query parameter for cache busting.

### Fixed

- Reduced stale frontend asset issues after upgrades by cache-busting key assets including `styles.css`, `version-badge.js`, and `pwa.js`.
- Prevented `Quick Start Guide` prompts from lingering on installs that already have active apps by clearing the pending onboarding flag automatically.
- Improved admin-only prompt behavior so non-admin/user-view sessions do not receive onboarding or release welcome modals.

### Migration Notes

- No schema/data migration required.
- Browser hard refresh should no longer be necessary for normal upgrades, but can still help if a service worker previously cached older assets.
- Existing installs with active apps should not see the `Quick Start Guide`; fresh/empty setups can use it as an onboarding path.

### Verification Checklist

- [ ] Admin on an empty/fresh setup sees `Quick Start Guide` with the sidebar-settings shortcut.
- [ ] Dismissing or using `Quick Start Guide` clears the prompt and it does not reappear unexpectedly.
- [ ] Admin sees the release `Welcome` modal for `v0.2.40` (with `Added/Changed/Fixed` highlights) when onboarding is not shown.
- [ ] Release `Welcome` modal opens the correct `v0.2.40` release notes link and only shows once per browser/version.
- [ ] Version badge current/latest pills link to GitHub release tags.
- [ ] Updated CSS/JS assets load after upgrade without stale script/style mismatches across Dashboard, App Overview, Launch, and Settings pages.
