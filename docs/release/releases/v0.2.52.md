# Changelog

## [v0.2.52] - 2026-03-29

### Added

- Added multi-arch Docker image publishing via GitHub Actions, producing `linux/amd64` and `linux/arm64` images from a single workflow run. Pushes to `main` tag `:development`; version tags publish both `:latest` and the semver tag.
- Added per-app custom HTTP headers so any additional headers (e.g. reverse-proxy auth tokens, vendor-specific headers) can be saved against an app and automatically merged into every outbound request Launcharr makes for that app.

### Changed

- All app API fetch paths now merge custom headers on top of their default auth headers, allowing custom values to override defaults where needed.
- Docker CI build arg now correctly resolves to the semver tag on releases and `development` on main branch pushes.

### Migration Notes

- No schema migration required.
- Existing installs will gain the Custom HTTP Headers field in Settings → Apps → [any app] → Connection after upgrade.
- Custom headers are opt-in and empty by default — no existing app config is affected.

### Verification Checklist

- [ ] Custom HTTP headers card appears in app connection settings form.
- [ ] Headers saved persist across settings saves and are reflected in outbound app requests.
- [ ] Docker image on Docker Hub shows both `amd64` and `arm64` manifests for `v0.2.52`.
- [ ] Version string in Launcharr shows `v0.2.52` (not `vmain` or `vdevelopment`) on tagged builds.
