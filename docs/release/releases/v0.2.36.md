# Changelog

## [v0.2.36] - 2026-02-22

### Added

- Added a `Hide scrollbars` theme option to reduce visible scrollbar rails while preserving scroll behavior.
- Added `Hide scrollbars` support to saved admin theme defaults and per-user theme preferences.

### Changed

- Updated the About page Discord support link to the current invite URL.

### Fixed

- Fixed theme default persistence so the new `Hide scrollbars` option can be saved and reapplied consistently.

### Security

- None.

### Deprecated

- None.

### Removed

- None.

### Migration Notes

- No schema/data migration required.
- Browser hard refresh is recommended after upgrade so updated theme assets/settings scripts are loaded.

### Verification Checklist

- [ ] `Settings -> Custom -> Themes` shows `Hide scrollbars` and saves correctly as admin default.
- [ ] User theme/profile settings preserve `Hide scrollbars` between sessions.
- [ ] Scrollbars are visually hidden when enabled, but page and carousel scrolling still work.
- [ ] `Settings -> About` Discord link opens the updated invite URL.
