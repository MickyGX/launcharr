# Changelog

## [v0.2.58] - 2026-04-12

### Summary

- Added Jellyfin synced login so users can sign into Launcharr directly with Jellyfin credentials from the main login page.
- Added Jellyfin user import and role management in Settings, alongside login visibility toggles for both Jellyfin and Plex.
- Fixed synced-role persistence across Plex/Jellyfin updates, improved login-page small-screen behaviour, and removed a sidebar collapsed-state flash during page load.

### Added

- Added a new Jellyfin synced login flow on `/login` that authenticates against Jellyfin's `/Users/AuthenticateByName` endpoint and creates a Launcharr session with Jellyfin-backed identity data.
- Added Jellyfin-aware session handling and theme preference keys so Jellyfin-synced users retain Launcharr preferences independently of local and Plex accounts.
- Added a `Jellyfin Users` settings card that fetches Jellyfin users from `/Users`, shows their recent activity and last Launcharr login, and allows admins to assign Launcharr roles from the UI.
- Added `Show on login screen` toggles to the Plex and Jellyfin settings forms so each provider can be exposed or hidden independently.

### Fixed

- Fixed role-saving behaviour so syncing roles from the Plex or Jellyfin settings tables merges provider-backed admins/co-admins instead of overwriting previously assigned entries from the other provider.
- Fixed the login experience on shorter displays by allowing the login layout to scroll, tightening provider-form spacing, and improving divider/button rendering for multiple auth methods.
- Fixed the saved collapsed-sidebar state briefly showing expanded accordion panels before client-side preferences initialize.

### Changed

- Provider-backed login methods now default to hidden until their Plex or Jellyfin app has actually been configured, while previously configured providers remain visible unless explicitly disabled.
- Local login now clears any stale Plex/Jellyfin session tokens before creating a Launcharr-local session, keeping auth-source transitions clean.
- Jellyfin API requests now support either API-key or session-token auth headers through shared request helpers.

### Upgrade Notes

- No schema migration is required. Update and restart Launcharr.
- If you want Jellyfin synced login, set the Jellyfin app URL in settings and optionally enable `Show on login screen` if you want it exposed immediately.
- Existing Plex/Jellyfin setups keep working. Older configured providers stay visible on the login page until you explicitly turn them off.
