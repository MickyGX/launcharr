# Launcharr v0.2.7

## [v0.2.7] - 2026-02-16

### Added

- Added `qBittorrent` default app (`Downloaders`) with icon support.
- Added `SABnzbd` default app (`Downloaders`) with icon support.
- Added downloader queue backend support for:
  - qBittorrent (`/api/v2/auth/login`, `/api/v2/torrents/info`)
  - SABnzbd (`/api?mode=queue&output=json&apikey=...`)
- Added queue item mapping/rendering for qBittorrent and SABnzbd in `downloaders-queue.js`.
- Added qBittorrent/SABnzbd support to combined and single download queue cards across dashboard and app overview.

### Changed

- Updated Combined Download Queue source selector UI to match the same pill/grid format used by other combined cards.
- Updated downloader card status filters to include richer states (`active`, `queued`, `completed`, `error`) where supported.

### Fixed

- Fixed Prowlarr API key field visibility in both:
  - `Settings -> Apps`
  - `/apps/prowlarr/settings`

### Migration Notes

- No schema migration required.
- If qBittorrent or SABnzbd is removed from sidebar/defaults, re-add from `Settings -> Custom -> Sidebar`.
- For qBittorrent queue data, provide username/password in app settings.
- For SABnzbd queue data, provide API key in app settings.

### Verification Checklist

- [ ] qBittorrent queue loads in app overview.
- [ ] SABnzbd queue loads in app overview.
- [ ] Combined Download Queue can include any of: Transmission, NZBGet, qBittorrent, SABnzbd.
- [ ] Combined downloader source selector uses card-style pill layout.
