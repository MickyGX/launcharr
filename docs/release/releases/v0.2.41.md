# Changelog

## [v0.2.41] - 2026-02-25

### Added

- Added local login rate limiting for `POST /login` with per-IP failure tracking, temporary lockout (`429`), and user-facing retry messaging after repeated failed attempts.
- Added auth-focused automated tests (`node:test` + `supertest`) covering login rate-limit helpers, login success/failure flows, and basic unauthenticated route guards.
- Added modular route files under `src/routes/` (`auth`, page routes, API routes, settings) to separate route registration from the main server bootstrap.
- Added shared EJS partials (`head`, `theme-init`, `nav-categories`, `user-menu`) and split page-specific stylesheets (`styles-auth`, `styles-home`, `styles-layout`, `styles-overview`, `styles-settings`) to reduce template/style duplication.

### Changed

- Refactored `src/index.js` to register route groups through `register*()` modules while preserving existing route paths and middleware behavior.
- Updated package scripts/dev tooling with a built-in `npm test` command and `supertest` dev dependency for HTTP route coverage.
- Reorganized frontend styling so common tokens/utilities remain in `public/styles.css` and page-specific rules load via additional CSS bundles.

### Fixed

- Fixed Romm iframe viewer launch cookie priming to improve embedded Romm launch reliability.
- Hardened log persistence writes by writing `logs.json` via a temporary file and rename to reduce partial/corrupt writes on interruption.

### Security

- Removed the insecure fallback session secret (`change-me`). When `SESSION_SECRET` is not set, Launcharr now generates a random secret at startup and logs a warning so operators can configure a persistent secret explicitly.

### Migration Notes

- No schema/data migration required.
- Set `SESSION_SECRET` in your runtime environment if you want sessions to persist across restarts (otherwise Launcharr will generate a new secret on each start and active sessions will reset).
- A hard refresh may help once after upgrade because page CSS is now split across multiple files.

### Verification Checklist

- [ ] `npm test` passes (`src/test/auth.test.js`).
- [ ] Local login returns `401` on invalid credentials and `429` after repeated failed attempts.
- [ ] Successful local login clears the rate-limit counter for that client IP.
- [ ] `Dashboard`, `Apps`, and `Settings` pages render with expected styling after the CSS split.
- [ ] Romm embedded (`iframe`) launch still succeeds when cookie priming is required.
- [ ] Logs continue to persist across restarts without malformed `data/logs.json`.
