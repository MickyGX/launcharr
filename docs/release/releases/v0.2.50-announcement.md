# Launcharr Release Announcement

## Headline

Launcharr `v0.2.50` is now available

## Short Summary

Launcharr `v0.2.50` adds a Curatorr widget, aligns Curatorr sidebar links with the current upstream layout, and fixes the missing Curatorr credential fields in Launcharr settings.

## What's New

- Added a `Curatorr` widget for all-time `Plex Users`, `Active Users`, and `Plays`.
- Added an admin-only Curatorr `Users` sidebar link that opens `/admin/users`.
- Updated Curatorr submenu ordering to match the current upstream sidebar layout.
- Restored Curatorr `Username` and `Password` fields on both Launcharr app-settings surfaces.

## Why It Matters

- Curatorr can now appear in widget bars with summary stats that match its admin user overview.
- Admins can jump straight into Curatorr user management from the Launcharr sidebar.
- Curatorr credentials can be configured where Launcharr actually expects them, so the widget and admin-route access are workable without manual config edits.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.50`
- Supplementary tag: `mickygx/launcharr:development`
- No migration steps required.
- Hard refresh the browser after upgrade so the updated Curatorr settings form and sidebar metadata are loaded.
- Save Curatorr local admin credentials in Launcharr app settings if you want the Curatorr widget to populate.
