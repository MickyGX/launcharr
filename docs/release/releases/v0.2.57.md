# Changelog

## [v0.2.57] - 2026-04-08

### Summary

- Added built-in Houndarr support as a default app with widget stats and credentials settings.
- Fixed the `slskd` widget ignoring username/password when a stale API key was stored, and added the missing API key field to both settings pages.
- Fixed a settings page flash where the General tab briefly appeared before the user's stored tab was activated.

### Added

- Added `houndarr` to `config/default-apps.json` as an opt-in default app in the Arr Suite category (port `8877`, launch mode `iframe`, multi-instance capable).
- Added Houndarr widget stats — Instances, Searches Today, and Found — fetched via session-cookie auth (`POST /login` → `GET /api/status`). Falls back to a health-only probe (`/api/health`) when no credentials are configured.
- Added Houndarr credentials card (username/password) to both the dedicated `app-settings` page and the inline settings panel in `settings`.

### Fixed

- Fixed `slskd` widget returning blank when an invalid or expired API key was stored. The widget now falls back to username/password token auth (`POST /api/v0/session`) when an API key returns `401`, rather than bailing out immediately.
- Fixed `slskd` API key field missing from both settings pages. The optional API key input is now shown alongside the existing username/password section on both the inline settings panel and the dedicated app-settings page.
- Fixed settings page briefly rendering the General tab before switching to the user's stored active tab on every page load, caused by `is-active` being hardcoded in the HTML on the General button and panel.

### Upgrade Notes

- No schema migration required. Update and restart Launcharr.
- If you use `slskd` with an API key: no action needed — the fallback to username/password is automatic. You can optionally clear the stale key from the new API key field in settings.
