# Changelog

## [v0.2.45] - 2026-03-05

### Added

- Added mobile launch-flow documentation media (`docs/media/launch-flow-mobile.gif`) and linked it from the wiki/media docs.

### Changed

- Updated carousel swipe handlers across overview modules to use pointer capture only for mouse drags (not touch pointers).
- Updated dashboard/settings horizontal widget rows to allow mixed gesture intent with `touch-action: pan-x pan-y`.
- Updated narrow-layout behavior for desktop touch devices so `.dash-main` is used as the vertical scrolling surface.

### Fixed

- Fixed Immich recently-added carousel swipe reliability by adding touch fallback listeners and horizontal-intent gesture checks.
- Fixed vertical page scrolling being blocked on desktop touchscreen devices when Launcharr is viewed in mobile-width layout, while preserving horizontal carousel swipes.

### Migration Notes

- No schema migration required.
- Rebuild/restart Launcharr and hard refresh browser assets once after upgrade.

### Verification Checklist

- [ ] Desktop touchscreen + mobile-width viewport: vertical page scrolling works on dashboard and app overview pages.
- [ ] Carousel left/right swipe still works across overview modules (including Immich).
- [ ] Mobile devices still support both vertical page scrolling and horizontal carousel swiping.
- [ ] Wiki home and docs media pages render both launch-flow GIF previews.
